Socket Service

Easiest way to use websockets without backend setup.

SocketService provides a super small library (<1Kb) that lets you connect to our websocket service, join a room for your app and emit your own events to all the sockets in your room.

HOW TO USE IT

First, include the SocketService library:

<script src="https://socketservice.herokuapp.com/SocketService.min.js"></script>

The SocketService library exposes a single global object SocketService with a method to initialize it, accepting 2 parameters:
- the name of the room you want to join.
- a callback function with a socket a parameter. 

SocketService.init( 'my-awesome-room', socket => {
    // do something with your socket
} )

The API for the socket is based on socket.io, exposing on and emit methods:
- the emit method lets you emit an event with the name your choice and an object for passing data. There is only one event that is fired automatically by the server that lets you know the socket joined the room 'room-joined'.
- the on method lets you handle your events and how all sockets in the room should use data. 

SocketService.init( 'my-awesome-room', socket => {
    socket.on( 'my-event', data => console.log( data ) )

    socket.on( 'room-joined', room => {
        console.log( `socket joined ${ room }` )

        socket.emit( 'my-event', { msg: 'hello test',  } )
    } )
} )
